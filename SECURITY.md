# Security policy

## Supported versions

Security fixes are considered for the current `main` branch and the latest published release.

## Reporting a vulnerability

Please do not open a public issue for a suspected security vulnerability.

Contact the repository owner through GitHub. If private vulnerability reporting is enabled for this repository, use the repository Security tab.

Include:

1. The affected repository and version or commit.
2. Steps to reproduce the issue.
3. Expected and observed behavior.
4. Any logs, proof-of-concept code, or screenshots that help verify the issue.

## Scope

This repository is part of an experimental WPVDB plugin suite. Reports are most useful when they describe a concrete impact on WordPress permissions, data exposure, remote execution, stored content, credentials, release integrity, or generated artifacts.

## Disclosure

Please allow time for triage and a fix before public disclosure.
